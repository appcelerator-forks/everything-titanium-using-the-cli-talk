Get / install the tool

  • Installing the CLI is an easy task: 
    $ npm install -g titanium
  • Want to live dangerously? 
    $ npm install -g git://github.com/appcelerator/titanium.git

$ titanium sdk

  • $ ti sdk helps manage the installed Titanium sdk:
    • list: list installed sdks
    • install: (un-)install a sdk version / update to newest
    • select: select a specific version by default

$ titanium config

  • ti config: list all options
  • ti config user.email: retrieves the value of a property
  • ti config user.email "hello@ticonf.org": sets property

Extending the CLI

Built extensible through plugins

  • add new commands
  • hook into exisiting commands

Creating new commands

  1. a single js file, where you want /path/to/sthing/pony.js
  2. register it for Titanium's cli: 
    $ ti config -a paths.commands /path/to/sthing
    (adds /path/to/sthing to the paths.commands config array)
  3. write the content of the command
    • a declarative part
      • command description, name, etc.
      • optionnaly, a config() method to set the default value of some execution option

Creating new commands

  • the content of the command
    • a validate() method called to validate the options passed to the command
    • a run() method, the "body" of the command
  • the command name is the filename
    • pony for /path/to/sthing/pony.js
    • not possible to rename/alias it - ticket + PR on their way :-)
    • the latter loaded takes precedence
      the last pony command overrides an earlier pony

Hooking into commands: concept

  • idea: run operations when certain events occur
  • hooks are a common concept, not specific to Titanium
  • can somehow be seen as "events" at the cli scale

Hooking into commands: howto

  1. a single js file, where you want /path/to/sthing/soundAfterBuild.js
  2. register it for Titanium's cli: 
    $ ti config -a paths.hooks /path/to/sthing
    (adds /path/to/sthing to the paths.hooks config array)
  3. write the content of the hook
    • using cli.on(), attach listeners to the occurence of hooks
    • cli.addHook() is an alias for cli.on()

A sample custom hook

var path = require('path'),
	play = require('play');

exports.cliVersion = '>=3.2';
exports.init = function(logger, config, cli, nodeappc) {
    cli.on('build.post.compile', function(builder, next) {
        play.sound(path.resolve(__dirname, 'sounds', 'trumpet.wav'));
        next();
    })
};

Some remarks about hooks

  • you can call hooks manually, using cli.emit: 
    /**
 * @param {String|Array} hookNames - The hook name or an array of many hook names
 * @param {Object} [data] - The event payload
 * @param {Function} callback - A callback when the event has finished firing
 */
function emit(hookNames, data, callback)
  • hooks callbacks can have a priority (default 1000): 
    cli.on('some.event', {
    priority: 999,
    post: function() { /* this is the callback */ }
});
    The lower the priority, the earlier the hook is executed

Some remarks about commands

  • do not forget to manually call the finished() callback: 
    exports.run = function (logger, config, cli, finished) {
    var i = 0;

    while (i < cli.argv.quantity) {
        console.log('ponies are flying!');
        i++;
    }

    if (typeof finished == 'function') {
        finished()
    }
};
    Else, the cli:post-execute hooks never gets called

package cli plugins

  • cli plugins are made of hooks and commands, and can be installed globally or locally
  • global install:
    • create the plugin somewhere, eg. /path/to/plugin
    • $ ti config paths.plugins -a /path/to/plugin
    • the plugin can contain commands and/or hooks
    • it will work in every Titanium project

package cli plugins

  • local install:
    • put the plugin in the plugins folder of your project
    • reference it in tiapp.xml:
    • may only contain hooks, not commands
    • usable hooks: the ones after cli:post-validate

ti-installr-hook

  • push your dev builds to Installr
  • install it: 
    $ npm install -g ti-installr-hook --unsafe-perm
  • Set token in tiapp.xml: 
    <property name="installr.api_token">ENTER_INSTALLR_API_TOKEN_HERE</property>
<property name="installr.notify" type="bool">true</property>
  • build the app: 
    $ ti build -p ios -T dist-adhoc --installr
  • and voila!

ti-testflight-hook

  • push your dev builds to Testflight (ios only). Do not use it. Go Installr.
  • install it: 
    $ npm install -g ti-testflight-hook --unsafe-perm
  • Set token in tiapp.xml: 
    <property name="testflight.api_token">ENTER_API_TOKEN_HERE</property>
<property name="testflight.team_token">ENTER_TEAM_TOKEN_HERE</property>
  • build the app: 
    $ ti build -p ios -T dist-adhoc --testflight
  • and voila!

alloy hooks into ti cli

the $ alloy new task installs this plugin: 
exports.installPlugin = function(alloyPath, projectPath) {
	var id = 'ti.alloy';

	// copy plugin
	var srcFile = path.join(alloyPath,'Alloy','plugin',CONST.PLUGIN_FILE);
	var destFile = path.join(projectPath,'plugins',id,CONST.PLUGIN_FILE);
	exports.copyFileSync(srcFile, destFile);

	// add the plugin to tiapp.xml, if necessary
	tiapp.init(path.join(projectPath, 'tiapp.xml'));
	tiapp.installPlugin({
		id: 'ti.alloy',
		version: '1.0'
	});
};
